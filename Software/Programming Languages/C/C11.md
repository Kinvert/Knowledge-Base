# C11

**C11** is the 2011 ISO C standard, formally **ISO/IEC 9899:2011**. It builds on [[C99]] and adds the first standard C memory model, atomics, optional threading support, compile-time assertions, type-generic selection, alignment features, anonymous structs/unions, and other cleanup features.

C11 is usually a good baseline for modern portable C when your compiler and libc support it. [[C17]] / C18 is mostly a bug-fix revision of C11, so many projects target C11 or C17 interchangeably.

---

## Quick Summary

C11 added:

- A language-level memory model.
- `_Atomic` and `stdatomic.h`.
- Optional `threads.h`.
- `_Static_assert`.
- `_Generic`.
- `_Alignas`, `_Alignof`, and `stdalign.h`.
- `_Noreturn` and `stdnoreturn.h`.
- Anonymous structs and unions.
- Better Unicode literal support.
- Feature-test macros for optional standard components.
- Optional bounds-checking interfaces in Annex K.

C11 also made some C99 features optional, including variable length arrays.

## Compiler Flags

Strict C11:

```bash
gcc -std=c11 -Wall -Wextra -Wpedantic file.c
clang -std=c11 -Wall -Wextra -Wpedantic file.c
```

GNU C11 dialect:

```bash
gcc -std=gnu11 file.c
clang -std=gnu11 file.c
```

C17/C18:

```bash
gcc -std=c17 -Wall -Wextra -Wpedantic file.c
clang -std=c17 -Wall -Wextra -Wpedantic file.c
```

`gnu11` means C11 plus GNU extensions. Use strict `c11` to check portability.

## Relationship To C99

C11 includes most C99 language features:

- Mixed declarations and code.
- `//` comments.
- Designated initializers.
- Compound literals.
- Flexible array members.
- `restrict`.
- `_Bool` and `stdbool.h`.
- `stdint.h`.
- `inline`.
- Variadic macros.

But C11 makes some features optional:

| Macro | Meaning |
|---|---|
| `__STDC_NO_ATOMICS__` | No standard atomics |
| `__STDC_NO_THREADS__` | No standard threads |
| `__STDC_NO_VLA__` | No variable length arrays |
| `__STDC_NO_COMPLEX__` | No complex arithmetic |

This means a C11 implementation is not always "C99 plus everything."

## Static Assertions

`_Static_assert` checks assumptions at compile time.

```c
#include <stdint.h>

_Static_assert(sizeof(uint32_t) == 4, "uint32_t must be 4 bytes");
```

Useful for:

- ABI assumptions.
- Struct layout expectations.
- Alignment requirements.
- Protocol/header sizes.
- Compile-time configuration checks.

Example:

```c
struct Header {
    uint32_t magic;
    uint16_t version;
    uint16_t flags;
};

_Static_assert(sizeof(struct Header) == 8, "wire header size changed");
```

## Atomics

C11 standardizes atomic types and operations through `_Atomic` and `stdatomic.h`.

```c
#include <stdatomic.h>

atomic_int counter = 0;

void tick(void) {
    atomic_fetch_add_explicit(&counter, 1, memory_order_relaxed);
}
```

Atomic object forms:

```c
_Atomic int a;
atomic_int b;
atomic_uint c;
atomic_bool flag;
```

Common operations:

```c
atomic_load(&counter);
atomic_store(&counter, 10);
atomic_fetch_add(&counter, 1);
atomic_exchange(&counter, 0);
atomic_compare_exchange_strong(&counter, &expected, desired);
```

## Memory Model

C11 defines how threads communicate through memory and what counts as a data race.

Important point:

**`volatile` is not a thread synchronization primitive.**

Use:

- Atomics for lock-free shared state.
- Mutexes/condition variables for protected shared state.
- Platform synchronization APIs when `threads.h` is unavailable.

Common memory orders:

| Order | Meaning |
|---|---|
| `memory_order_relaxed` | Atomicity only; no ordering synchronization |
| `memory_order_acquire` | Prevent following reads/writes from moving before the load |
| `memory_order_release` | Publish prior writes before the store |
| `memory_order_acq_rel` | Acquire + release for read-modify-write ops |
| `memory_order_seq_cst` | Strongest default global ordering |

Typical stop flag:

```c
#include <stdatomic.h>
#include <stdbool.h>

atomic_bool stop_requested = false;

void request_stop(void) {
    atomic_store_explicit(&stop_requested, true, memory_order_release);
}

bool should_stop(void) {
    return atomic_load_explicit(&stop_requested, memory_order_acquire);
}
```

Relaxed counter:

```c
atomic_uint events = 0;

void record_event(void) {
    atomic_fetch_add_explicit(&events, 1, memory_order_relaxed);
}
```

Use `memory_order_relaxed` only when ordering does not matter.

## Threads

C11 added `threads.h`, but it is optional and not universally available.

```c
#include <threads.h>
#include <stdio.h>

int worker(void *arg) {
    puts((const char *)arg);
    return 0;
}

int main(void) {
    thrd_t t;

    if (thrd_create(&t, worker, "hello") != thrd_success) {
        return 1;
    }

    thrd_join(t, NULL);
    return 0;
}
```

`threads.h` includes:

- `thrd_t`
- `thrd_create`
- `thrd_join`
- `mtx_t`
- `mtx_lock`
- `mtx_unlock`
- `cnd_t`
- `cnd_wait`
- `tss_t`
- `call_once`

Practical note:

- On POSIX systems, [[Pthreads]] are still more common.
- On Windows, Win32 threads or a portability wrapper may be more common.
- `stdatomic.h` is usually more important than `threads.h`.

## Generic Selection

`_Generic` provides compile-time type dispatch.

```c
#include <stdio.h>

#define type_name(x) _Generic((x), \
    int: "int", \
    float: "float", \
    double: "double", \
    default: "other")

int main(void) {
    printf("%s\n", type_name(3.14));
}
```

Useful for:

- Type-generic math wrappers.
- Debugging helpers.
- Safer macros.
- Lightweight generic APIs without C++ templates.

Example:

```c
#define abs_value(x) _Generic((x), \
    int: abs, \
    long: labs, \
    long long: llabs, \
    float: fabsf, \
    double: fabs \
)(x)
```

## Alignment

C11 adds standard syntax for alignment.

```c
#include <stdalign.h>

struct alignas(64) CacheLineCounter {
    int value;
};

_Static_assert(alignof(struct CacheLineCounter) == 64,
               "expected cache-line alignment");
```

Core forms:

```c
_Alignas(64) char buffer[1024];
size_t a = _Alignof(double);
```

Header convenience macros:

```c
#include <stdalign.h>

alignas(16) float v[4];
size_t a = alignof(float);
```

Use cases:

- SIMD buffers.
- Cache-line padding.
- DMA buffers.
- ABI-sensitive structs.
- Lock-free data structures.

## `_Noreturn`

`_Noreturn` tells the compiler that a function never returns.

```c
#include <stdnoreturn.h>
#include <stdlib.h>

noreturn void fatal(void) {
    abort();
}
```

Without `stdnoreturn.h`:

```c
_Noreturn void fatal(void);
```

Useful for:

- Fatal error handlers.
- Panic functions.
- Infinite dispatch loops.
- Exit wrappers.

## Anonymous Structs and Unions

C11 standardizes anonymous structs/unions.

```c
struct Vec4 {
    union {
        struct {
            float x, y, z, w;
        };
        float data[4];
    };
};

struct Vec4 v;
v.x = 1.0f;
v.data[1] = 2.0f;
```

Common in embedded, hardware register definitions, graphics/math types, and ABI overlays.

Use carefully: layout and aliasing rules still matter.

## Unicode Literals

C11 adds better literal prefixes:

```c
char *s = u8"UTF-8 text";
char16_t *s16 = u"UTF-16 text";
char32_t *s32 = U"UTF-32 text";
```

Types are available through:

```c
#include <uchar.h>
```

Practical support varies by compiler/libc and by what you expect to do with the encoded text.

## Annex K Bounds-Checking Interfaces

C11 includes optional bounds-checking functions in Annex K, such as:

```c
strcpy_s(...)
memcpy_s(...)
sprintf_s(...)
```

These are controversial and not widely portable across Unix-like systems. Do not assume Annex K exists just because the compiler supports C11.

If portability matters, prefer:

- `snprintf`
- explicit length tracking
- well-tested project-local wrappers
- safer higher-level parsing APIs

## Variable Length Arrays In C11

C99 requires VLAs. C11 makes them optional.

```c
#ifdef __STDC_NO_VLA__
#error "This project requires variable length arrays"
#endif
```

Even when available, many projects ban VLAs because runtime stack allocation can be dangerous.

Prefer fixed maximums:

```c
enum { MAX_ITEMS = 256 };
int tmp[MAX_ITEMS];
```

or heap allocation:

```c
int *tmp = malloc(n * sizeof *tmp);
if (!tmp) {
    return;
}

free(tmp);
```

## Example C11 Program

```c
#include <stdatomic.h>
#include <stdbool.h>
#include <stdint.h>
#include <stdio.h>

struct Header {
    uint32_t magic;
    uint16_t version;
    uint16_t flags;
};

_Static_assert(sizeof(struct Header) == 8, "unexpected header size");

static atomic_uint packets_seen = 0;

static void record_packet(void) {
    atomic_fetch_add_explicit(&packets_seen, 1, memory_order_relaxed);
}

int main(void) {
    struct Header h = {
        .magic = 0x12345678u,
        .version = 1,
        .flags = 0,
    };

    record_packet();
    printf("magic=%08x packets=%u\n",
           h.magic,
           atomic_load_explicit(&packets_seen, memory_order_relaxed));

    return 0;
}
```

## Portability Checks

Check whether C11 optional pieces are missing:

```c
#ifdef __STDC_NO_ATOMICS__
#error "C11 atomics are required"
#endif

#ifdef __STDC_NO_THREADS__
#warning "C11 threads unavailable; using platform threads"
#endif

#ifdef __STDC_NO_VLA__
#define PROJECT_HAS_VLA 0
#else
#define PROJECT_HAS_VLA 1
#endif
```

Check C version:

```c
#if __STDC_VERSION__ >= 201112L
/* C11 or newer */
#endif
```

Common values:

| Macro Value | Standard |
|---|---|
| `199901L` | C99 |
| `201112L` | C11 |
| `201710L` | C17/C18 |

## When To Use C11

Use C11 when:

- You want standard atomics.
- You care about defined multithreaded memory behavior.
- You want compile-time assertions.
- You want standard alignment syntax.
- Your compiler/libc support is modern enough.
- You are writing new portable systems code.

Be cautious when:

- You depend on `threads.h`.
- You depend on Annex K functions.
- You assume VLAs exist.
- You target old embedded compilers.
- You need compatibility with old MSVC or strict vendor C compilers.

Practical default for new C projects:

```bash
-std=c11 -Wall -Wextra -Wpedantic
```

or:

```bash
-std=c17 -Wall -Wextra -Wpedantic
```

C17 is usually preferable if available because it is C11 with defect fixes.
